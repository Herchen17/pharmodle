2. Three-Tier Answer System

Implemented in both the game frontend (checkGuess function) and the puzzle data schema.

Tier 1 — Correct ✅ Exact match, brand name alias, equivalence group, or parent term. Full points awarded.
Tier 2 — Acceptable Alternative ✅ A different drug that fits ALL 5 clues equally well. Full points awarded. Shows: "You guessed [X] — that's also correct! The target answer was [Y]."
Tier 3 — Near Miss ⚠️ Same drug class, but at least one clue distinguishes the answer. Counts as wrong guess. Shows: "So close! [Educational feedback about the distinction]."
Tier 4 — Incorrect ❌ No match. Standard wrong guess behaviour.

Key design decisions:

  • Near miss = breaks streak (it's wrong, just with helpful feedback)
  • Acceptable alternatives capped at 3 per puzzle (more = bad puzzle)
  • Canonical answer always shown on game end
  • Each puzzle's tiers assessed individually during generation

4. Generation Pipeline

1
Haiku Generation — Generates puzzles in batches of 5. Each puzzle includes self-rated confidence (1-5). Uses AMH-derived drug list with class-mates for three-tier assessment. ~140 API calls for 700 puzzles.
↓
2
Auto-Grading — 12 heuristic rules run locally (no API cost). Catches ~60% of issues. Separates into GOOD/ACCEPTABLE (pass through) and NEEDS_WORK/POOR (go to Sonnet).
↓
3
Sonnet Review — Reviews flagged puzzles + low-confidence ones. Checks factual accuracy, funnel quality, three-tier classification. Verdicts: PASS, REWORK, REJECT. ~100-150 API calls.
↓
4
Human QA (You) — HTML review report with approve/flag/notes for each puzzle. Cross-check pharmacology against AMH. Final quality gate.
